# Security Policy

## Supported Versions

Security fixes are applied to the current `main` branch and the latest published npm package.

## Reporting A Vulnerability

If you believe you found a security issue:

1. do not open a public issue first
2. use GitHub private vulnerability reporting if available
3. include:
   - affected package or path
   - reproduction steps
   - impact summary
   - dependency versions when relevant

We will triage the report, confirm impact, and prepare a fix before public disclosure when possible.
