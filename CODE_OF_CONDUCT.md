# Code of Conduct

## Our Standard

This project is intended to be useful to contributors, users, and maintainers working across frontend tooling, agent workflows, and open source skill development.

Expected behavior:

- be respectful and specific
- critique code and decisions, not people
- prefer reproducible reports over vague complaints
- keep discussions useful for the next contributor who reads them

Unacceptable behavior:

- harassment or personal attacks
- discriminatory language
- bad-faith issue filing or review behavior
- intentionally misleading technical claims

## Enforcement

Project maintainers may remove comments, close discussions, or block contributors when behavior damages the working environment.

For serious issues, open a private report through GitHub security advisories or the repository contact channels.
